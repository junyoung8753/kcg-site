# KCG KRX API Approval Runbook

Last updated: 2026-05-06 KST.

This runbook covers the approval-first path for using KRX/Koscom gold-market data in KCG. It does not authorize payment, public redistribution, search launch, secret storage, or automatic trading behavior.

## Current Decision

- Use path: `approval first`.
- Company email: use `kcgoldx@gmail.com`. A Gmail address is not by itself the blocker for a business application; KRX distinguishes the applicant by legal/business registration for 법인 use and personal verification/social login for 개인 use.
- Production rule: KRX data stays blocked from KCG public/reference price surfaces until KRX/Koscom approval, public/commercial display scope, attribution wording, and third-party distribution limits are confirmed.
- Current code guard: if `MARKET_DATA_PROVIDER=krx`, `krx-open-api`, `krx-openapi`, or `koscom` is set before approval, the site returns the existing fallback market data and exposes the blocked state through `/api/health`. It does not call KRX and does not overwrite company posted prices.

## Official Evidence To Recheck

- KRX Open API service method: `https://openapi.krx.co.kr/contents/OPP/INFO/OPPINFO003.jsp`
  - 법인은 사업자 정보 등록, 개인은 본인인증 또는 소셜 로그인.
  - 인증키 신청 and API 활용 신청 both require administrator approval before use.
- KRX Open API terms: `https://openapi.krx.co.kr/contents/OPP/INFO/OPPINFO002.jsp`
  - The terms define API users as approved individuals or corporations.
  - API use is stated as non-commercial, third-party provision is restricted, and screens using KRX statistical information must state that fact unless KRX sets a different display rule.
  - A key is limited to 10,000 requests per day and can be deleted after 12 months of non-use.
- KRX service list: `https://openapi.krx.co.kr/contents/OPP/INFO/service/OPPINFO004.cmd`
  - `금시장 일별매매정보` is listed under 일반상품.
- KRX Data Marketplace distribution products: `https://openapi.krx.co.kr/contents/OPP/DATA/OPPDATA002.jsp`
  - Real-time, end-of-day, and reference market-information distribution are treated as data products.
- KRX data purchase guidance: `https://openapi.krx.co.kr/contents/OPP/DATA/OPPDATA001.jsp`
  - Paid data download/use is reviewed after payment and use-purpose review; 법인명 입금 can require a separate form process.
- Koscom market data service: `https://english.koscom.co.kr/eng/main/contents.do?menuNo=300102`
  - Koscom distributes market information generated by Korea Exchange/KOFIA to institutions, information providers, and media through real-time and batch channels.
- KRX Gold Market safety guidance: `https://open.krx.co.kr/contents/OPN/01/01050206/OPN01050206.jsp`
  - KRX Gold Market is a securities-company account based exchange-traded market and is separate from private homepage/pre-deposit/OTC physical trades.

## Legal Alternative Review - 2026-05-06

There is no safe "shortcut" that should be implemented as production code before approval. The reviewed legal paths are:

- Approved KRX Open API path: use `금시장 일별매매정보` only after authentication key issuance, API utilization approval, attribution wording, request limits, and public/commercial-display scope are confirmed.
- Contract/data-product path: use KRX data purchase or Koscom market-data service if KRX/Koscom says public web display, redistribution, real-time, delayed, or derived formula use requires a contract.
- Interim reference path: keep the existing Gold API/Metals.Dev-style reference provider boundary, link-only KRX education, and KCG human operator posted-price workflow.
- Blocked workaround path: do not call hidden KRX web endpoints, OTP/download URLs, public-page scraping, browser automation, TradingView extraction, competitor price copying, or "copy then offset" formulas.

If approval is internal-only, Codex may add an admin/operator reference importer later, but public pages must still show company posted prices first and must not publish raw KRX data.

## Human-Only Steps

Junyoung or a company representative must complete only the service-forced steps:

- Log in or sign up with `kcgoldx@gmail.com`.
- If applying as 법인, register the business information for `한국센터금거래소`; if KRX requires domain email, representative verification, phone/email verification, terms consent, document upload, or legal representative confirmation, complete those directly.
- In the use-purpose field, use this safe wording unless KRX asks for a stricter format:

```text
한국센터금거래소 내부 시세 산정 참고용. KRX 금시장 데이터를 직접 거래가격으로 표시하지 않고, 회사 고시 시세 산정 참고 데이터로 사용하며, 출처/기준시각/회사 고시 시세와의 구분을 표시합니다. 공개 표시 또는 재배포가 필요한 경우 승인 범위에 따릅니다.
```

- Report only non-secret approval facts back to Codex: approval result, internal-only/public-display scope, required attribution text, request limits, valid period, whether a paid contract is required, and support contact.
- Do not paste API keys, passwords, MFA codes, recovery codes, OAuth tokens, cookies, service-role keys, or payment/card values into chat/docs/Git.

## Codex Implementation Boundary After Approval

Codex can proceed only after non-secret approval scope is known:

- Add a KRX provider behind an explicit env/provider gate.
- Candidate env names are placeholders until the official API specification is confirmed: `KRX_OPEN_API_KEY`, `KRX_OPEN_API_SERVICE_ID`, and `MARKET_DATA_PROVIDER=krx`.
- Parse the approved KRX response into source metadata and KCG formula input, not directly into company posted prices.
- Store suggestion/history metadata with provider name, source updated time, approval scope note, and formula snapshot.
- Keep public UI hierarchy as `회사 고시 시세` first, `KRX/Koscom 참고 데이터` second or admin-only if approval scope says internal use only.
- Keep fallback behavior: KRX failure, missing approval/env, rate limit, stale data, or ambiguous terms must not hide posted prices and must not publish raw KRX values.

## Stop Conditions

Stop and classify as user-only or blocked if any of these appear:

- Payment, paid data contract, card entry, or 법인 계약 signature is required.
- KRX/Koscom says public web display, redistribution, commercial use, or formula-derived publication is not allowed.
- KRX requires exact attribution or display format not yet represented in the UI.
- Approval is only internal-use; then use KRX only for admin/operator reference, not public pages.
- Any secret value is needed; Codex should prepare the Vercel env flow only after company project access exists.
