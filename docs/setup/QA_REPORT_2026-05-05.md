# KCG QA Report - 2026-05-05

## Design Thesis

KCG should feel like a Jongno gold-exchange price desk: posted prices and product categories first, operational copy second, and admin screens that behave like a control console rather than a marketing page.

## QA Scope

- Public routes: `/`, `/prices`, `/products`, `/services`, `/company`, `/about`, `/announcements`.
- Admin routes: `/admin`, `/admin/prices`, `/admin/products`, `/admin/announcements`, `/admin/launch`.
- Viewports: mobile 390px and desktop 1440px.
- Evidence: Playwright tests, rendered/source audit, screenshot set, and manual screenshot review.

## Issues Found And Fixed In This Pass

- `/admin/prices` automatic-price switch could submit the opposite mode because the click handler changed React state before native form submission. Fixed by submitting explicit `nextIsAutoOn` hidden values and adding a source audit guardrail.
- `/admin` still looked like a light-skinned dark dashboard rather than a practical operations console. Fixed by rebuilding it around daily tasks, posted-price status, automatic-price review queue, product visibility, notices, launch blockers, and direct action links.
- `/admin/prices` still had a dark warning box and older automatic-price wording. Fixed by converting warnings to light admin styling and using `자동시세` wording consistently.
- Desktop home price table could still feel vertically tight in the first viewport. Fixed by reducing price-panel typography/spacing and increasing the desktop hero working height so all posted-price rows are visible.
- `국제 현재가` and domestic conversion rows were technically correct but too small for a price-checking site. Fixed by increasing row body, price, unit, and change text sizes.
- Mobile `/products` first viewport used too much vertical space before the catalog controls. Fixed by tightening the hero height, top padding, catalog spacing, and sort-control rhythm.
- Mobile `/services` first viewport stacked the three-step flow too deeply. Fixed by making the core steps a compact three-column mobile row and reducing the image/header height.
- `/admin/prices` current public price snapshot used repeated category labels such as `순금시세`, which made buy/sell meaning less immediate. Fixed by mapping rows to `순금 살 때`, `순금 팔 때`, `백금 살 때`, and similar operator-facing labels.
- `/admin` showed `Demo` as a storage label. Fixed by changing it to `미리보기 저장` and using consistent Korean date formatting for the posted time.
- Production mobile could register the price-panel close tap before hydration finished, leaving the panel open and hiding the restore control. Fixed by showing the close button only after hydration.
- `/admin/prices` automatic-price switch could show a saved OFF state while the screen preview was ON, which read as contradictory. Fixed by separating `저장된 운영 상태` from `화면 상태`.
- Admin screenshot evidence did not include `/admin` and `/admin/products`. Fixed by adding opt-in admin dashboard and products screenshots to the screenshot helper.
- A regression test now checks that mobile `/products` reaches the catalog controls in the first viewport.

## v0.2.8 Admin Console Follow-Up

- Root-cause panel: the ON switch problem was a real bug, not user confusion. The form submitted state-derived hidden inputs after local state changed; the fix submits the intended next state directly.
- Representative/operator panel: `/admin` now starts with the work that matters today: posted prices, automatic-price status, review queue, product visibility, announcements, and launch blockers.
- Price operator panel: `/admin/prices` keeps automatic and manual modes visually distinct and removes leftover dark warning styling.
- Evidence panel: source audit now requires the `nextIsAutoOn` submission pattern and forbids the old `onModeChange(!isAutoOn)` pattern from coming back.

## v0.2.7 Admin Console Follow-Up

- Representative/operator panel: the admin console now uses a light theme so status cards, forms, and warnings read like an operations tool rather than a dark marketing page.
- Price operator panel: automatic price ON/OFF is a switch-like control, and the screen no longer relies on confusing saved-state/screen-state wording.
- Price operator panel: `지금 계산 실행` shows applied/held/failure results near the automatic operation panel, so an operator can tell why a price did or did not change.
- Product/content manager panel: key admin forms now show pending labels such as `저장 중...`, `계산 중...`, and `반영 중...`.
- Evidence panel: admin screenshots now include manual and automatic `/admin/prices` states, including a mobile automatic-state capture.

## v0.2.6 Expert-Panel Follow-Up

- Public/customer panel: home price disclosure remains the first decision surface; mobile product and service paths now reveal the working controls earlier.
- Representative/operator panel: admin price rows now use explicit buy/sell labels and avoid generic repeated metal names.
- Market/data panel: Gold API and TradingView remain secondary reference surfaces; company posted prices stay the source of truth.
- Design/mobile panel: image-led route headers are still present, but mobile first viewports now spend less height before action/content.
- Launch panel: robots/noindex, sitemap block, no payment/cart/trading, and no competitor scraping remain unchanged.

## v0.2.2 Expert-Panel Follow-Up

- Added `docs/setup/AUTO_PRICE_OPERATIONS_BRIEF.md` so the company can understand that daily automatic checks are the free/Hobby-plan posture, while hourly or 30-minute automatic checks require Vercel Pro or an external scheduler.
