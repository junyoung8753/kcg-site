- Move tasks to `done` only when the acceptance criteria and verification note are true.
- Verified KCG site source/UI changes should be pushed and deployed for live review by default after the required QA gate. Keep new DNS/domain policy changes, robots/noindex release, search indexing, payment/trading behavior, secrets/env changes, and hard-to-reverse infrastructure changes blocked until junyoung gives explicit approval.
