## Findings

### P0-1 Admin Server Action Authorization

- User harm: an abnormal request path could mutate posted prices, products, or notices outside the visible admin page flow.
- Business risk: company posted prices and public product information are core commercial surfaces.
- Repro path: compare `/admin/*` middleware protection with mutating Server Actions in `src/actions/*`.
- Fix: added `requireAdminActionSession()` and call it at the top of mutating price, product, and announcement actions.
- Verification: `npm run audit:site` now requires the helper and action calls.

### P0-2 Automatic Price OFF Was Not A True Kill Switch

- User harm: staff could think automatic pricing is OFF while stale-manual logic turns it back on later.
- Business risk: company posted prices may change without explicit operator intent.
- Repro path: `runPriceAutoRefresh()` previously enabled auto publish when manual updates were stale.
- Fix: automatic price OFF now remains OFF. Stale manual registration produces warnings only.
- Verification: `npm run audit:site` blocks the old auto-enable pattern and ON-switch copy.

### P1-1 Product List Reference Prices Needed Caveats

- User harm: exact-looking product card amounts can be read as final sale/buying quotes.
- Fix: product cards now show `참고가입니다. 현장 확인 후 최종 안내합니다.` and sort labels say `참고가 낮은순/높은순`.
- Verification: targeted Playwright product catalog test passed.

### P1-2 Launch Dashboard Needed User-only Blockers

- User harm: an operator could mistake noindex review readiness for public search launch readiness.
- Fix: `/admin/launch` now exposes final product price/fee confirmation, image final-use approval, final admin password rotation, and ownership-transfer decision as user-only launch blockers.
- Verification: targeted Playwright admin launch test passed.

### P1-3 Mobile `/about` Needed First-viewport Map Action

- User harm: a customer opening the location route had to scroll before seeing map actions.
- Fix: the top route section now includes Naver Map, Kakao Map, and head-office phone actions.
- Verification: targeted Playwright mobile `/about` first-viewport test passed.

### P1-4 `/admin/login` Showed Admin Shell Before Login

- User harm: the primary login task was below the fold and the unauthenticated screen exposed console chrome.
- Fix: middleware passes `x-kcg-path`; admin layout renders `/admin/login` without the admin shell.
- Verification: targeted Playwright admin login test passed.

### P2-1 Image Alt/Social Wording Was Too Strong

- User harm: screen-reader/social contexts could read representative images as final stock/product proof.
- Fix: home/social alt text now says representative KCG-derived consultation imagery instead of final real-product proof.
- Verification: `npm run audit:site` requires the safer wording.
